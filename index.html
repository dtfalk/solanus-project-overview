Mimicker 1 - Sanitization

Solanus text to LLM pairs

‚è± Estimated time: 3-4 weeks

This step takes original writings from Father Solanus and rewrites them in a plain, neutral style. That neutral version is created by an LLM, which is a large language model or smart text program.

The neutral version and the original are stored as pairs. Later, those pairs teach a model how to move from neutral language back into the unique style of Father Solanus.

  • Input - chunks of Solanus writings
  • Output - pairs of neutral and original text
  • Use - training data for the style model

Plain language explanation

The project is built as a set of small tools that work together. Each tool can also be used on its own. The steps below retell the README in everyday language and explain technical terms along the way.

Where we are

We currently have the publicly available writings of Father Solanus Casey in PDF format. A PDF is a digital version of a document that looks like a scanned page or a printed book.

Where we want to go

Our goals are:

  • Let a computer speak in his voice and style
  • Have that computer answer questions by using his real writings
  • Present all of this through an interactive hologram of Father Solanus

Key ideas in simple terms

  • Machine readable - text that is clean and organized so a computer program can understand it.
  • LLM - stands for large language model. It is a smart text program that can read, write and summarize.
  • JSON - a tidy way of storing information using name and value pairs, like labeled boxes in a closet.
  • Metadata - small bits of information about a document, like its date, type and who it was written to.
  • Index - a search friendly catalog that lets us quickly find the right documents.
  • Embedding - a list of numbers that acts as a fingerprint of a piece of text, used for smart searching.
  • GPU - a graphics processing unit, a type of computer chip that is very good at doing many calculations in parallel.

The rest of the page and the detailed README below explain how the extraction, indexer and mimicker connect. Even if the full hologram is not used, the extraction pipeline is already useful for archivists, the indexer is already a digital library and the mimicker is already a style transformation tool.
